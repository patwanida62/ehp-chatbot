Version 1.0 Base URL: /api Format: JSON Date: 11 May 2026

1. ภาพรวมการทำงาน 2. Authentication 3. POST /api/tickets — สร้าง Ticket ใหม่ 4. GET /api/tickets/:id — ดึงข้อมูล Ticket 5. PATCH /api/tickets/:id/status — อัปเดตสถานะ 6. Status Codes 7. Error Response Format
1. ภาพรวมการทำงาน

LINE Bot จะเรียก API ของระบบ Ticket ใน 3 กรณี:

กรณีEndpoint ที่เรียกเมื่อไหร่
ลูกค้าพิมข้อความที่ตรง keyword POST /api/tickets เปิด Ticket อัตโนมัติทันที
ลูกค้าพิม "ตรวจสอบ #001" GET /api/tickets/001 ดึงสถานะล่าสุดตอบกลับ
เจ้าหน้าที่ปิด Ticket POST /notify/ticket-closed ระบบ Ticket เรียก Bot เพื่อแจ้งลูกค้า
⚠️ Bot จะเรียก TICKET_API_BASE_URL ที่กำหนดใน .env เช่น https://your-system.com/api
2. Authentication

ปัจจุบัน Bot ไม่ได้ส่ง Authorization Header ไปด้วย ถ้าระบบ Ticket ต้องการ Auth ให้แจ้งทีม Bot เพื่อเพิ่ม Header เช่น:

"Authorization": "Bearer YOUR_API_KEY"
"X-API-Key": "YOUR_API_KEY"
3. สร้าง Ticket ใหม่
POST /api/tickets Bot เรียกเมื่อลูกค้าพิมข้อความที่ตรง keyword
Request Body
FieldTypeRequiredคำอธิบาย
title string REQUIRED ข้อความที่ลูกค้าพิม เป็นหัวข้อปัญหา
lineUserId string REQUIRED LINE User ID ของลูกค้า ใช้ส่งแจ้งเตือนกลับ
reporterName string OPTIONAL ชื่อ Display Name ของลูกค้าใน LINE
service string OPTIONAL ชื่อ Service เช่น "EHP CIS"
category string OPTIONAL หมวดหมู่ปัญหา
subCategory string OPTIONAL หมวดหมู่ย่อย
ตัวอย่าง Request
{
  "title": "เข้าระบบไม่ได้",
  "lineUserId": "U0df970281236ea540eef7b92ff5ab406",
  "reporterName": "PAT",
  "service": "EHP CIS",
  "category": "ปัญหาการใช้งานทั่วไป",
  "subCategory": "การใช้งานทั่วไป"
}
Response — 201 Created
FieldTypeคำอธิบาย
idstringเลข Ticket ที่ระบบสร้างให้
titlestringหัวข้อปัญหา
reporterNamestringชื่อผู้แจ้ง
servicestringชื่อ Service
categorystringหมวดหมู่
subCategorystringหมวดหมู่ย่อย
resolutionstringวิธีแก้ไข (ใส่ "" ถ้ายังไม่มี)
createdAtstring (ISO 8601)วันเวลาที่รับเรื่อง
ตัวอย่าง Response
{
  "id": "1001",
  "title": "เข้าระบบไม่ได้",
  "reporterName": "PAT",
  "service": "EHP CIS",
  "category": "ปัญหาการใช้งานทั่วไป",
  "subCategory": "การใช้งานทั่วไป",
  "resolution": "",
  "createdAt": "2026-05-11T10:00:00.000Z"
}
4. ดึงข้อมูล Ticket
GET /api/tickets/:id Bot เรียกเมื่อลูกค้าพิม "ตรวจสอบ #001"
Path Parameter
ParameterTypeคำอธิบาย
idstringเลข Ticket ID
Response — 200 OK
FieldTypeคำอธิบาย
idstringเลข Ticket
titlestringหัวข้อปัญหา
statusstringสถานะ: open / in_progress / closed / cancelled
resolutionstringวิธีแก้ไข (ถ้ายังไม่มีให้ส่ง "" หรือ null)
updatedAtstring (ISO 8601)วันเวลาที่อัปเดตล่าสุด
ตัวอย่าง Response
{
  "id": "1001",
  "title": "เข้าระบบไม่ได้",
  "status": "in_progress",
  "resolution": "ทีมงานกำลังตรวจสอบ",
  "updatedAt": "2026-05-11T11:30:00.000Z"
}
5. อัปเดตสถานะ Ticket
PATCH /api/tickets/:id/status เรียกจาก Bot หรือระบบ Ticket เพื่ออัปเดตสถานะ
Request Body
FieldTypeRequiredค่าที่รับได้
status string REQUIRED open / in_progress / closed / cancelled
ตัวอย่าง Response — 200 OK
{
  "id": "1001",
  "status": "closed",
  "updatedAt": "2026-05-11T14:00:00.000Z"
}
💡 เมื่อปิด Ticket ให้ระบบ Ticket เรียก Bot ด้วย:
POST http://<bot-server>/notify/ticket-closed
Body: { "ticketId": "1001", "title": "...", "lineUserId": "U..." }
Bot จะส่งข้อความแจ้งลูกค้าใน LINE อัตโนมัติ
6. Status Values
ValueความหมายBot แสดงผล
open เปิดแล้ว รอดำเนินการ 🟡 เปิดแล้ว รอดำเนินการ
in_progress กำลังดำเนินการ 🔵 กำลังดำเนินการ
closed ดำเนินการเสร็จสิ้น 🟢 ดำเนินการเสร็จสิ้น
cancelled ยกเลิกแล้ว 🔴 ยกเลิกแล้ว
7. Error Response Format

เมื่อเกิดข้อผิดพลาด ให้ส่ง HTTP Status Code ที่ถูกต้องพร้อม body:

{
  "error": "ข้อความอธิบายข้อผิดพลาด"
}
HTTP Statusใช้เมื่อ
400Request ไม่ถูกต้อง เช่น ขาด field
401Authentication ล้มเหลว
404ไม่พบ Ticket ID ที่ระบุ
500Server error ภายใน
