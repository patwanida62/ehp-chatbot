การทำงานของระบบ
ลูกค้าทำอะไร Bot ทำอะไร API ที่เรียก สถานะ
พิม keyword เช่น "เข้าระบบไม่ได้" เปิด Ticket + ส่งใบรับเรื่องกลับ POST /tickets ✅ พร้อม
พิม "ตรวจสอบ #001" ดึงสถานะ + วิธีแก้ + ผู้ดำเนินการ GET /tickets/:id ✅ พร้อม
— รับ Webhook แจ้งเตือนปิด Ticket POST /notify/ticket-closed ✅ พร้อม
— รับ Webhook แจ้งเตือนเปิด Ticket POST /notify/ticket-opened ✅ พร้อม
สิ่งที่ Dev ต้องทำในระบบ Ticket
🔌
เพิ่ม 3 API Endpoint ในระบบ Ticket เดิม
ดูรายละเอียดใน API-Document.html
Endpoint Method หน้าที่ ความสำคัญ
/api/tickets POST สร้าง Ticket ใหม่จาก LINE Bot บังคับ
/api/tickets/:id GET ดึงสถานะ + วิธีแก้ไข + ผู้ดำเนินการ บังคับ
/api/tickets/:id/status PATCH อัปเดตสถานะ + วิธีแก้ไข + ผู้ดำเนินการ บังคับ
💡 Response ต้องมี field: id, title, status, resolution, resolvedBy, resolvedAt, createdAt, updatedAt
ดูรายละเอียดครบถ้วนใน API-Document.html
⚠️ เมื่อเจ้าหน้าที่ ปิด Ticket ในระบบ ให้ระบบ Ticket เรียก Bot ด้วย:
POST http://<bot-server>:3000/notify/ticket-closed
Body: {"ticketId":"001", "title":"...", "lineUserId":"U..."}
