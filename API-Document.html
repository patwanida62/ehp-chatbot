📡 API Document — EHP LINE Chatbot

เอกสาร API สำหรับการเชื่อมต่อระหว่าง LINE Bot ↔ Ticket System

🤖 Bot Server: port 3000 🎫 Ticket API: port 4000 📅 อัปเดต: พฤษภาคม 2569

สารบัญ

🤖 BOT SERVER (port 3000)
① POST /webhook/line ② POST /notify/ticket-closed ③ POST /notify/ticket-opened
🎫 TICKET API (port 4000)
④ POST /tickets — สร้าง Ticket ⑤ GET /tickets/:id — ดึงข้อมูล ⑥ PATCH /tickets/:id/status — อัปเดต ⑦ GET/POST /keywords ⑧ GET/PUT /settings
🤖 Bot Server localhost:3000 Webhook + Notification endpoints
POST /webhook/line รับข้อความจาก LINE Platform (LINE → Bot)
🔒 ใช้ HMAC-SHA256 verify signature อัตโนมัติ — LINE Platform เรียก endpoint นี้เองทุกครั้งที่ลูกค้าส่งข้อความ ไม่ต้อง Dev เรียกเอง

POST /notify/ticket-closed Ticket System แจ้ง Bot เมื่อปิด Ticket
⚠️ Ticket System ต้องเรียก endpoint นี้ทุกครั้งที่ปิด/อัปเดตสถานะ Ticket เพื่อให้ Bot แจ้งลูกค้าใน LINE

Request Body

FieldTypeRequiredที่มา (จากฟอร์ม)
ticketIdstringบังคับรหัส Ticket
titlestringบังคับหัวข้อปัญหา
lineUserIdstringบังคับLINE User ID ลูกค้า (เก็บตอนสร้าง Ticket)
statusstringแนะนำสถานะ → "เสร็จสิ้น"
resolvedBystringแนะนำผู้แก้ปัญหา
resolutionstringแนะนำวิธีแก้ไข
causestringไม่บังคับสาเหตุ
resolvedDatestringแนะนำวันที่เสร็จสิ้น (YYYY-MM-DD)
resolvedTimestringแนะนำเวลาเสร็จสิ้น (HH:MM)
remarkstringไม่บังคับหมายเหตุ

ตัวอย่าง Request

POST http://bot-server:3000/notify/ticket-closed
Content-Type: application/json

{

  "resolvedTime": "10:58",

}

POST /tickets Bot เรียกเพื่อสร้าง Ticket ใหม่อัตโนมัติ

Request Body (Bot ส่งมา)

FieldTypeRequiredคำอธิบาย
titlestringบังคับเรื่อง = ข้อความลูกค้า
detailstringแนะนำรายละเอียด = ข้อความลูกค้า
lineUserIdstringบังคับLINE User ID (เก็บไว้เพื่อส่งข้อความกลับ)
reporterNamestringชื่อผู้แจ้ง (LINE Display Name)
servicestringService จาก Settings
categorystringCategory จาก Settings
subCategorystringSub Category จาก Settings
slastringSLA จาก Settings เช่น "4H"
problemDatestringวันที่พบปัญหา (YYYY-MM-DD)
problemTimestringเวลาที่พบปัญหา (HH:MM)

Response ที่ต้องส่งกลับ (Bot ต้องการ)

{
  "id":          "26003615",         // รหัส Ticket (required)
  "title":       "เข้าระบบไม่ได้",   // หัวข้อ
  "status":      "in_progress",      // สถานะเริ่มต้น
  "resolution":  "",                  // ว่างตอนสร้าง
  "resolvedBy":  "",                  // ว่างตอนสร้าง
  "resolvedAt":  null,               // null ตอนสร้าง
  "updatedAt":   "2026-05-12T10:33:00.000Z"
}
✅ หลังจาก POST Bot จะเรียก GET /tickets/{id} ทันทีเพื่อดึงข้อมูลเต็ม กรุณาส่ง id กลับมาให้ครบ
GET /tickets/:id Bot เรียกเพื่อดึงข้อมูล Ticket เต็ม (ใช้ 2 กรณี)
Bot เรียก endpoint นี้ 2 กรณี: (1) หลัง POST สร้าง Ticket และ (2) เมื่อลูกค้าพิมพ์ "ตรวจสอบ #id"

Response ที่ต้องส่งกลับ

{
  "id":          "26003615",
  "title":       "เข้าระบบไม่ได้",

ค่าBot แสดงเป็น

closed🟢 ดำเนินการเสร็จสิ้น
cancelled🔴 ยกเลิกแล้ว

PATCH /tickets/:id/status Dashboard อัปเดตสถานะ Ticket

Request Body

{
  "status":      "closed",            // open | in_progress | closed | cancelled
  "resolution":  "แก้ไข config แล้ว",  // วิธีแก้ไข
  "resolvedBy":  "วันดา ภารสถิตย์"   // ผู้แก้ไข
}
เมื่อ status = "closed" ระบบต้อง set resolvedAt เป็น timestamp ปัจจุบันด้วย

Response

{ // Ticket object ที่อัปเดตแล้ว
  "id": "26003615", "status": "closed",
  "resolvedAt": "2026-05-12T10:58:00.000Z", ...
}
GET /keywords ดึงรายการ Keywords POST /keywords เพิ่ม Keyword DELETE /keywords/:index ลบ Keyword
Bot โหลด Keywords ทุกครั้งที่ได้รับข้อความ — Keywords ที่ match จะสร้าง Ticket อัตโนมัติ รองรับการจับคู่แบบใกล้เคียง (bidirectional)

GET Response / POST Body

GET /settings ดึงค่า Settings PUT /settings อัปเดต Settings

Settings Object

ตัวแปรในข้อความ (Message Templates)

ตัวแปรค่าที่ใส่
{id}รหัส Ticket
{title}หัวข้อปัญหา
{status}สถานะ (แปลเป็นภาษาไทยแล้ว)
{resolution}วิธีการแก้ไข
{resolvedBy}ผู้ดำเนินการแก้ไข
{updatedAt}วันเวลาที่อัปเดตล่าสุด
{resolvedAt}วันเวลาที่แก้ไขเสร็จ (ว่างถ้ายังไม่ปิด)
